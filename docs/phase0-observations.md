# Phase 0 Observations

## 我們做出了什麼？

- 擷取並檢視 `src/fixtures/phase-0/messy-reports.json`（M-001 至 M-012），確認每筆原始資料的 `rawText`、`sourceType`、`verificationStatus` 與 `updatedAt` 可見性。
- 執行專案格式化與測試，確認開發環境可用（`pnpm format`、`pnpm test`）。

## 哪些資料讓 agent 很容易整理？

- 有明確數量或狀態描述的回報（例如 M-010 提供雨鞋剩餘數量與盤點時間）。
- 明確標註為現場志工回報或 volunteer_update，且帶時間戳的項目，可作初步摘要。

## 哪些資料讓 agent 亂補或誤判？

- 社群貼文與截圖（例如 M-004、M-005）缺乏可驗證的出處與日期，agent 若不受限容易把「未標示日期的截圖」當成當日公告。
- 第二手轉述（例如 M-006、M-011）常缺少是否經當事人同意公開地址或細節，agent 可能把未經同意的地點視為可派遣的任務。

## 哪些資訊不能直接變成任務？

- 未提供具體且可驗證地址或當事人同意的需求（例如 M-011 的家具搬運）。
- 來源來自電話或遠端家屬而非現場確認，無法確認當事人狀態（例如 M-012）。
- 時效或出處不明的截圖或社群貼文（例如 M-005），不應直接作為道路封閉或行動的依據。

## 哪些資訊的操作者不是當事人？

- M-006：志工轉述學校側門情況（可能非房屋當事人）。
- M-011：志工代長者轉述（當事人無法直接操作手機）。
- M-012：外地家屬來電（來電者不在現場，資訊為代述）。

## 我們剛剛偷偷假設了什麼？

- 假設某些回報的時間為「最新狀態」而非過去某時點。
- 假設社群貼文或群組留言為可立即採信的來源。
- 假設轉述的地點可公開且可執行（未確認當事人同意）。

## 這次復盤看到的三個資料品質問題

### 重複上傳

- 同一個事件可能被不同人用不同說法重複回報，例如同一個物資點、集合點或人力需求被社群貼文、志工更新、現場回報各寫一次。
- 風險是後續整理者可能把一件事看成多件事，造成重複派人、重複送物資，或把已過期資訊又重新放進流程。
- 需要人工確認的是：這些回報是不是同一件事、哪一筆時間最新、哪一筆來源比較接近現場。

### 偽造資料

- 有些資訊看起來像公告、截圖或轉傳內容，但缺少發布者、日期、原始連結或可追溯來源。
- 風險是使用者可能因為格式看起來正式，就把它當成可靠資訊，尤其是道路封閉、集合點開放、物資數量這類會影響行動的內容。
- 需要人工確認的是：來源是否真的存在、內容是否被改過、是否已過期、是否有可信的人或單位可以確認。

### 訊息模糊

- 有些原始資訊只說「老雜貨店後面」「那邊」「先不要再派人」或「疑似需要協助」，缺少明確地點、時間、當事人與需求範圍。
- 風險是 agent 或整理者會為了讓資料看起來完整，自動補出原文沒有的地點、需求類型或下一步。
- 需要人工確認的是：到底是哪個位置、誰需要協助、需求是否仍存在、是否同意被聯繫或公開資訊。

## 如果要讓這個工具變可靠，下一步需要什麼？

- 建立人工查核流程：記錄誰確認、確認時間、確認結果（accept / reject / need more info）。
- 要求來源驗證欄位（例如截圖出處、回報者聯絡方式、是否經當事人同意公開地址）。
- 在 UI 顯著標示 `verificationStatus`（`needs_review` / `unverified`）且禁止顯示為已確認。

## 我們需要更清楚資料格式的地方

- 明確區分 `sourceType`（官方公告 / 現場回報 / 社群貼文 / 電話）與其可接受的信任等級。
- 加入 `consentToShareAddress`、`reportedByContact`、`originalTimestamp` 等欄位以便判斷是否可派工。

## 我們需要人工確認的地方

- M-001：需確認「老雜貨店後面」是否為可公開與可到達的地址。
- M-002 / M-004 / M-010：需核實現場物資庫存與下一次盤點時間，避免重複運送。
- M-005：需確認截圖來源與日期，是否為官方公告。
- M-011：需確認長者是否同意公開地址與是否同意志工進入家中搬運家具。
- M-012：需核實來電者身份與被幫助者的具體位置與同意情況。

## 示範草稿範例（已初始化於工作台）

- M-001（示範候選，`confidence: low`）
  - humanReviewNote: "人工質疑：來源未確認；不應直接派工。需聯絡回報者核實時間與位置。"
  - blockers: ["來源未完全核實；時間或地點可能不明"]
  - suggestedNextStep: `send_to_human_review`
  - 說明：此筆為社群貼文類型，原文地點描述模糊（老雜貨店後面），缺乏可驗證地址與發布時間，故標為需人類核實。

- M-002（示範候選，`confidence: medium`）
  - humanReviewNote: "人工質疑：需再次盤點與確認是否為當日現場狀態。"
  - blockers: ["需人工確認來源/細節"]
  - suggestedNextStep: `send_to_human_review`
  - 說明：志工更新提及雨鞋庫存，但時間點不明；示範為中等信心候選，仍需現場盤點確認。
