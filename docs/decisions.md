# 需求取捨決策

> 這份文件用來記錄 v1 的優先服務對象與取捨決策，並留出人類判斷痕跡。

## 我決定 v1 優先服務的使用者

- [ ] 回報者
- [x] 資訊整理者
- [ ] 行動者

原因：

- 目前 prototype 的主要價值在於讓資訊整理者把混亂原始資訊轉成後續可理解的候選資料，並避免把不確定內容誤導成行動任務。
- 優先服務資訊整理者可降低因未確認資訊導致行動失誤的風險。

## 這個使用者最想完成的事

- 判斷哪些資訊可以相信、哪些需要人工確認。
- 把來源、原文、狀態一起保存，避免誤把不確定資訊當成已確認。

## 這個使用者最怕什麼錯誤

- 模糊資料被自動變成任務。
- AI 補了原文沒有的細節，讓整理結果看起來比原始資料更確定。

## 我今天暫時不處理的需求

- 讓回報者自己填表並直接送出任務。
- 自動派工或直接指派現場行動者的功能。

## 我不採用的 AI 建議

- AI 建議：把缺少的地點、數量或需求類型自動補成「可行動任務」。
  - 不採用原因：這會讓未確認資訊變成假象的已確認內容，違反 v1 的保守資訊整理目標。

## 必須保留人工確認的判斷

- 是否已確認信息可行動。
- 是否原始來源足夠清楚。
- 是否有重複、衝突或可能過期的資訊。

## 今天明確不做什麼

- 不做自動任務派工或行動決策。
- 不做讓 AI 自動判定「這筆資料可以直接執行」的功能。

## 我仍不確定的地方

- 如何在 UI 中最清楚區分「原始資訊」「候選資訊」「已確認資訊」。
- 是否需要針對回報者與行動者顯示不同的視圖。
